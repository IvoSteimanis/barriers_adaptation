**Authors:** Ivo Steimanis & Max Burger, Philipps University Marburg

## Software Requirements

- **Stata 19** (backward compatible to Stata 16)
- All required user-written packages are included in `scripts/libraries/stata/` and are automatically loaded by `run.do`. No manual package installation is needed.

## Data Sources

Four raw data files are stored in `data/`:

| File | Description |
|------|-------------|
| `bangladesh2018.csv` | Study 1: Household survey on sea-level rise perceptions and motivated reasoning (Bangladesh, 2018) |
| `bangladesh2021.xlsx` | Study 2: Household survey on hazard exposure, sunk cost bias, and mobility preferences (Bangladesh, 2021) |
| `bangladesh2021_translations.xlsx` | Open-ended response translations for Study 2 |
| `bangladesh_GPS.dta` | External input: Bangladesh reference sample from the Global Preferences Survey (Falk et al., 2018) for risk and patience norms |

## Instructions

1. Open Stata
2. Set the working directory to the folder containing `run.do`
3. For a clean run, delete the `processed/` and `results/` folders
4. Run `run.do`

The master script auto-detects the working directory, sets up required packages, and executes all cleaning and analysis scripts sequentially. A timestamped log file is saved to `scripts/logs/`.

## Script Descriptions

Scripts are executed in the following order by `run.do`:

| Order | Script | Description |
|-------|--------|-------------|
| 1 | `01_clean_Bangladesh2018.do` | Cleans Study 1 raw data (`bangladesh2018.csv`) → `processed/bangladesh2018.dta` |
| 2 | `02_clean_Bangladesh2021.do` | Cleans Study 2 raw data (`bangladesh2021.xlsx`) → `processed/bangladesh2021.dta` |
| 3 | `risk_time_analysis.do` | Appends Studies 1 & 2 with GPS reference sample; imputes missing risk/patience items (MI, 20 imputations); constructs composite risk-taking and patience indices within each imputation → `processed/risk_time_mi.dta` (full MI dataset) and `processed/risk_time_rdy.dta` (single extraction + GPS for plots) |
| 4 | `03_analysis.do` | Generates all tables and figures reported in the manuscript and Supplementary Online Material (SOM) |

All packages are bundled in `scripts/libraries/` and ready to use.

## Multiple Imputation

Study 2 uses multiple imputation (MI) with chained equations (predictive mean matching, k=5 nearest neighbors, 20 imputations) for missing values in stay-reason variables. Stage 2 regression analyses (sunk-cost sensitivity) use `mi estimate` to combine coefficient estimates across all 20 imputations via Rubin's rules.

The risk/patience composite construction in `risk_time_analysis.do` also uses MI (20 imputations) for missing staircase and survey items. Composites are built within each imputation: z-scores, GPS-weighted indices, and min-max normalization are computed separately for each of the 20 imputed datasets. The full MI dataset is saved as `risk_time_mi.dta`. Stages 3–4 regression tables (S9 and S10) use `mi estimate` to combine coefficient estimates across all imputations via Rubin's rules, properly accounting for imputation uncertainty.

A single extraction (imputation 20) is saved as `risk_time_rdy.dta` with the GPS reference sample appended, used only for descriptive plots (kernel densities, subgroup mean comparisons, Kruskal-Wallis tests).
